# 架构

本文档总述 KiCloud 当前版本的系统定位、技术栈、目录结构、核心模块、页面入口、请求链路和主要业务边界。

---

## 1. 项目定位

KiCloud 是一个面向个人或小团队的轻量级私有网盘项目，当前版本采用单体 Spring Boot 应用实现，已实现：

1. 用户认证与权限管理
2. 单文件上传、分片上传与下载
3. 分享链接生成与管理
4. 后台用户权限管理

当前版本采用前后端一体架构。

---

## 2. 技术栈

### 2.1 后端

- Java
- Spring Boot
- Spring MVC
- Spring Security
- Spring Data JPA
- Hibernate
- SQLite
- Lombok

### 2.2 页面层

- Thymeleaf 模板
- 静态 CSS / JavaScript
- 服务端渲染 + 页面内调用 JSON 接口

### 2.3 存储层

- 数据库存储：SQLite（`./data/kicloud.db`）
- 文件内容存储：本地磁盘（`./data/storage`）
- 分片临时文件：本地磁盘临时目录（默认 `./data/temp`）

## 3. 运行配置

配置文件位置：`src/main/resources/application.properties`

当前关键配置包括：

| 配置项 | 当前值 | 说明 |
|---|---|---|
| `server.servlet.session.timeout` | `30m` | Session 超时时间 |
| `spring.datasource.url` | `jdbc:sqlite:./data/kicloud.db` | SQLite 数据库路径 |
| `spring.jpa.hibernate.ddl-auto` | `none` | 数据库初始化策略 |
| `spring.servlet.multipart.max-file-size` | `10GB` | 单次上传大小上限 |
| `spring.servlet.multipart.max-request-size` | `10GB` | 请求体大小上限 |
| `kicloud.storage.root` | `./data/storage` | 文件实际存储根目录 |
| `kicloud.init.superadmin.username` | 配置项指定 | 默认超级管理员用户名 |
| `kicloud.init.superadmin.password` | 配置项指定 | 默认超级管理员初始密码 |

### 3.1 默认超级管理员初始化

相关代码位置：

- `src/main/java/club/muimi/kicloud/KicloudApplication.java`

如果数据库中不存在 `SUPERADMIN`，则自动创建一个默认超级管理员账号。

初始化时会：

1. 使用配置文件中的默认用户名和密码
2. 对密码执行哈希编码后保存
3. 将该用户角色设置为 `SUPERADMIN`
4. 赋予`1024TB`的默认总空间额度

创建的超级管理员账号用户名和密码分别依据`kicloud.init.superadmin.username`和`kicloud.init.superadmin.password`。

如果存在`SUPERADMIN`，则不会进行任何操作。

### 3.2 当前部署形态特征

当前版本系统默认是：

- 单体应用
- 单机本地文件存储
- 单 SQLite 数据库文件
- 基于 Session 的会话管理

因此它更适合：

- 本地开发
- 单机部署
- 小规模自用或演示环境

而不是直接作为多实例共享存储架构使用。

## 8. 权限结构

当前角色包括：

- `USER`
- `ADMIN`
- `SUPERADMIN`

### 8.1 USER

- 使用文件管理功能
- 创建和管理自己的分享链接
- 无权访问 `/admin/**`

### 8.2 ADMIN

- 具备管理员后台访问能力
- 可生成邀请码
- 可管理普通用户
- 不具备超级管理员级别的全站视角

### 8.3 SUPERADMIN

- 具备 ADMIN 的全部能力
- 可查看与删除全站分享链接
- 可浏览和搜索其他用户文件
- 可进行更高等级的角色管理
- 应用启动时若系统中不存在超级管理员，会自动初始化一个默认超级管理员

## 9. 认证与放行策略

### 9.1 Session + Spring Security 双轨协同

当前系统不是完全依赖 Spring Security 默认登录流程，而是：

1. `UserController` 接收登录请求
2. `UserService` 手动校验用户名 / ID 与密码
3. 把 `LoginUser` 写入 Session
4. 同步写入 `SPRING_SECURITY_CONTEXT`
5. 后续请求既通过 Spring Security 的路由限制，也通过 MVC Interceptor 再次校验 Session 合法性

### 9.2 公开放行路径

当前显式放行的主要路径包括：

- `/user/login`
- `/user/logout`
- `/user/register`
- `/css/**`
- `/js/**`
- `/images/**`
- `/webjars/**`
- `/error`
- `/forbidden`
- `/link/**`

其中 `/link/**` 的放行保证了公开分享页与匿名下载能力。

## 10. 数据与持久化结构

### 10.1 数据库存储内容

当前数据库主要保存：

- 用户信息
- 邀请码信息
- 文件 / 文件夹元数据
- 分享链接元数据

### 10.2 磁盘存储内容

当前磁盘主要保存：

- 上传后的真实文件内容

目录树本身并不依赖物理目录树完整映射，而是主要由数据库中的 `parentId` 关系维护。
